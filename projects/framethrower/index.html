Framethrower

At the current state I cannot go into detail of exactly what Framethrower is, however I can allude that it is an animation community with an integrated workflow with the animation software. I have currently been working on it for some time and I can briefly go over the technology stack. I will write interesting aspects of the development process in the blog section. For now, here is a full list of the technologies we utilize.

Workflow

  • Lerna - The main mono repo is managed by Lerna
  • Github Actions - is utilized for CI/CD

Cloud & Infrastructure

  • AWS - Is the backbone of our cloud infrastructure
  • Terraform - Is the tool utilised to plan and provision cloud infrastructure
  • Docker - Is used for building images which are deployed to our ECR service on AWS. We also utilize docker-compose for local development

Backend

  • PostgresQL - Is not only our database, but also utilises PL/pgSQL as our main approach to our business logic
  • Serverless - Is a framework to deploy lambda functions, which we utilise for heavy lifting functionality
  • Sqitch - Is a database migration tool
  • PostGraphile - Is a node middleware which build our GraphQL API from introspecting the PG Schema

Frontend

  • React - JS Framework
  • Apollo Client - GraphQL and state management
  • Styled Components - CSS injected components JSX
  • Material UI - Is a library of useful and well styled javascript components
  • Webpack - Is our Javascript bundler we use to build our front end application
Maya Integration
  • Maya CMDS - We utilize the Python wrapper of Mayas MEL commands
  • PyQt - Framework for user interfaces in Python
