Review Player

JavaScript was not built for handling video, and nothing like building a media player demonstrates that. Currently studios rely on desktop media players for reviewing shot and asset development, and while it works, it can introduce challenges when teams are distributed across locations, and on different sets. Incoorperating a web application in the internal review process, seems like a natural progression. Especially since browsers included the requestAnimationFrame, and the canvas element, which has made it all the more plausible.

Request Animation Frame is a feature of modern browsers that lets you run javascript functions, before the next frame is rendered. I use it to validate that the current time of the played video is within the frame range specified in the user interface. Another pretty cool feature of the review player is that you can write on notes on top of the frame, (meanwhile the mirror function of the player is respected).

The Review Player is part of framethrower which is not open source. I will make sure to link to the application when it is online.

The Review Player is written in JavaScript using React and Material UI for interface elements.
